- Member 
  -  사용자에대한 테이블
  -  아이디와 이름을 갖고있음
- Subject
  - 특강 주제에 대한 테이블
  - 특강 이름과 강사에대한 정보가 있음
- Lecture
  - 실제 특강 테이블
  - 주제에대해 실제 특강이 날짜가 언제인지, 현재 남은 신청 인원이 몇인지 정보가 있음
- MemberLecture
  - 사용자와 특강 매핑 테이블
  - 사용자가 신청한 특강에 대한 정보를 갖고있음
 
# 동시요청에 대한 처리 방식
- 30명으로 제한된 특강에 선착순으로 요청한 사용자에 대한 수강신청 처리를 한다
- Lecture 에 대한 레코드를 조회할때 비관적 Lock을 걸어 Lecture에 대한 수정을 하나의 요청만 처리하도록 한다
- 수강신청 요청이 Lecture의 capacity를 감소시킨다
- 만약 capacity가 0이라면 더이상 신청하지 못한다

# 유저가 같은 강의를 중복 신청못하도록
- MemberLecture를 조회해 이미 해당 강의에 신청한 유저라면 중복 신청하지 못하도록 처리
- 이 내용도 위의 Lock 영향을 받기에 동시 처리에대한 방지가 되어있다
